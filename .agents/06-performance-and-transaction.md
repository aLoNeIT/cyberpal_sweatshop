# 性能与事务规范

本文档用于统一事务使用边界、慢请求判定和高并发场景下的处理要求。

## 1. 事务使用规范

- 事务要善用但不滥用。
- 高并发业务应通过削峰、解耦等方式保证最终一致性。
- 事务边界应放在 `Logic` 层或专门的 `Transaction` 类中，禁止在 `Controller` 层直接开启或提交事务。
- 事务内禁止执行不可回滚的外部副作用，例如短信发送、文件写入、远程接口调用、队列投递后无补偿处理等。
- 涉及多表状态流转时，必须在事务内校验当前状态和影响行数，避免重复提交、并发覆盖或越权更新。
- 长事务必须拆分或重构，避免在事务中执行慢查询、循环查询、外部调用和大批量处理。

## 2. 慢任务与异步处理

- 单次请求耗时超过 10 秒的任务，应交由后台异步执行，由前端异步获取结果。
- 异步任务、队列和后台进程的运行时约束见 `13-hyperf-runtime-standards.md`。

## 3. 性能阈值

- SQL 执行超过 3 秒视为慢 SQL，必须优化。
- PHP 执行超过 5 秒视为慢执行，必须优化。
