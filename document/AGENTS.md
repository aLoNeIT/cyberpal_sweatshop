# document/AGENTS.md

本目录是 `cyberpal_sweatshop` 项目的 SQL 与产品文档配套目录，用于维护数据库初始化脚本、基础数据和 PRD 文档。处理本目录时，应将其视为数据定义和产品文档目录，而不是应用源码目录。

## 项目索引

- `sql/`：数据库初始化 SQL、字典、菜单、功能、基础数据。进入该目录工作前必须阅读 `sql/AGENTS.md`。
- `prd/`：产品需求文档目录（待补充）。新增 PRD 时按应用端分类维护。

## 目录职责

### `sql/`

`sql/` 是数据库资产目录，主要维护完整初始化脚本。新增或修改 SQL 时，优先维护 `sql/initialization/` 下的正式初始化脚本；不要把正式变更落在临时目录。

SQL 目录的具体结构、初始化脚本顺序、字典规则、菜单功能编码和验证要求见 `sql/AGENTS.md`。

### `prd/`

`prd/` 是产品需求文档目录（待补充），按应用端分类保存 Markdown 文档。

## 通用维护规则

- 避免无关格式化，尤其不要整文件重写大 SQL、字典 SQL 或静态数据文件。
- 保留现有文件编码和换行风格。多数 SQL 文件包含中文内容，读取或编辑时需要注意乱码风险。
- 新增资产应放在最接近用途的现有目录中，并遵循该目录的 `AGENTS.md` 规则。
- 不提交凭据、主机专属连接串、生产数据导出或不应进入源码仓库的敏感信息。
- 对外共享或提交前，检查本次变更是否包含授权、账号、连接信息或环境私有内容。

## 验证要求

- 文档或 SQL 修改后，至少运行针对性 `git diff -- <changed files>`。
- 涉及目录重命名或文件迁移时，运行 `git status --short` 确认 Git 识别的新增、删除和重命名范围符合预期。
- 涉及 SQL 结构、字典、菜单或权限数据时，按 `sql/AGENTS.md` 的验证要求执行。

## 提交

提交信息使用清晰的祈使句，说明本次变更目标。提交变更时必须遵守仓库 Git 配置和忽略规则；被 `.gitignore` 忽略的文件、目录不要添加到 Git 索引或提交中。
