## 1. 产品定位

### 一句话愿景
让每个用户都能在几分钟内创建、配置并调度属于自己的 AI Agent，像使用水电一样按需调用 OMP 原生多 Agent 能力与工具生态。

### 核心价值主张
- **低门槛**：直接复用 OMP 原生 MCP 管理、Profiles 配置隔离，用户无需自研调度内核。
- **可复用**：历史会话持久化保留、归档、续聊（resume）/ 分叉（fork），沉淀为个人上下文资产。
- **可计量**：token 输入/输出/缓存与估算费用透明可见，首版聚焦「用量展示」，不做扣减/余额。

### 目标用户
- **本版**：需要把 AI Agent 嵌入业务流的个人开发者 / 技术运营 / 业务专家（单用户租户）。
- **后续**：平台运营者（Admin）。

### 角色
- **EndUser（终端用户）**：注册、登录、创建配置 agent、发起会话、查看历史与用量。首版核心角色。

| **R-006** | 历史会话列表与续聊（resume）/分叉（fork） | 会话可续聊/分叉启动新会话并继承上下文 | ✅ |

## 4. 功能模块

### 4.1 用户账号体系

## 5. 关键产品决策

| 决策 | 结论 |
|------|------|
| 注册方式 | 仅邮箱 + 密码 |
| 会话模型 | 单 Agent 单会话 |
| 计费模式 | 首版仅用量展示（不扣减） |
| Skill 来源 | 平台托管库（无用户上传） |
| 归档策略 | 统一不可恢复；活跃 ≤100 条 / 归档 ≤50 条；超限拒绝创建不清理；手动归档二次确认；自动归档默认 30 天（Admin 可调） |
| 协作范围 | 纯单用户，无团队/组织 |
| 鉴权方案 | Session + Redis（token 请求头优先，Cookie 兜底） |
| 多设备登录 | 允许，不互踢 |
| Agent 导入导出 | 不做 |
