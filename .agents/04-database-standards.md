# 数据库规范

本文档用于统一项目数据库表结构、字段命名、索引命名和字段设计要求。

## 1. 基础命名规范

- 表名全部小写。
- 字段名全部小写。
- 表前缀统一使用项目简写。
- 模型类 `$table` 只填写业务表名，不包含数据库连接配置中的表名前缀，例如配置前缀为 `cs_` 时，模型中写 `user`，不写 `cs_user`。
- 表名为组合词时，使用下划线分割。
- 字段前缀应为表名缩写。
- 字段名为组合词时，使用下划线分割。
- 字段命名以英文单词为主，特殊情况可使用拼音缩写。

## 2. 业务表必备字段

业务表必须包含以下字段：

- 自增 `id`
- `create_time`
- `update_time`
- `delete_time`

说明：

- 时间字段必须使用 `int` 或 `bigint` 存储秒级 Unix 时间戳。
- 禁止业务表时间字段使用 `datetime`、`timestamp` 或字符串日期格式。
- 模型需要自动维护创建、修改、删除时间时，必须在子类中按实际字段名显式定义 `CREATED_AT`、`UPDATED_AT`、`DELETED_AT`。

## 3. 索引命名规范

- 联合索引：`un_idx_字段名`
- 唯一索引：`uniq_字段名`
- 非唯一索引：`idx_字段名`

## 4. 常用字段语义规范

- `phone`、`mp`：移动电话
- `telephone`、`tel`：座机
- `account`：账号
- `content`：详细描述，一般为 `text` 类型
- `img_xxx`：图片类型字段
- `ed` 结尾字段：布尔类型字段，例如 `opened`
- `state`：一般用于整型状态字段
- `status`：一般用于字符型字段，代表外键关联码表
- `time` 结尾：单位为秒级时间戳
- `date` 结尾：表示当天 0 点的时间戳

## 5. 字段设计规范

- 字段类型必须符合业务需求。
- 所有字段必须存在默认值。
- 所有字段禁止为 `null`。
- 时间字段统一使用秒级 Unix 时间戳，字段类型只能使用 `int` 或 `bigint`，并设置为 `unsigned` 类型。
- 金额字段统一使用 `decimal(10,2)` 设计。
