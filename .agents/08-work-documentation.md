# 工作文档规范

本文档用于统一需求文档和任务文档的编写内容要求。

## 1. 需求文档

- 按照统一格式编写，可按实际情况增加内容节点。
- 必须包含流程示意图。
- 必须包含 UI 交互图。
- 必须包含界面元素内容数据标准。
- 需求描述必须包含以下模板节点，暂无内容时也要保留节点并写明"无"或"待确认"：

```text
需求概述：

功能说明：

前置条件：

后置条件：

业务流程：

交互界面：

备注：
```

- `需求概述：` 说明需求背景、目标用户、业务目标和本次需求边界。
- `功能说明：` 按功能点拆解说明页面、接口、数据、权限、校验和异常场景要求。
- `前置条件：` 说明使用该功能前必须存在的账号、角色、配置、基础数据或业务状态。
- `后置条件：` 说明功能完成后产生的数据变化、状态变化、通知、日志、审计或后续业务影响。
- `业务流程：` 按步骤描述主流程、分支流程和异常流程；需要图示时应补充流程示意图。
- `交互界面：` 说明入口、页面元素、按钮行为、列表字段、表单字段、弹窗、提示文案和 UI 交互图。
- `备注：` 记录限制条件、依赖外部系统、待确认事项、历史兼容要求和其他补充说明。

## 2. 任务文档

- 按照统一格式编写，可按实际情况增加内容节点。
- 必须包含技术开发内容描述。
- 必须包含准确的工时评估。
- 任务描述必须包含以下模板节点，暂无内容时也要保留节点并写明"无"或"待确认"：

```text
任务概述：

功能描述：

所属模块：

功能入口：

依赖项：

影响范围：

输出成果：
```

- `任务概述：` 简述任务背景和要解决的问题。
- `功能描述：` 按技术实现视角描述涉及的接口、页面、数据变更和业务逻辑。
- `所属模块：` 标明任务所属的业务模块或系统模块。
- `功能入口：` 描述功能的菜单入口、路由地址或页面位置。
- `依赖项：` 列出前置依赖的任务、接口、数据或配置。
- `影响范围：` 说明本任务涉及的代码文件、数据库、配置和接口范围。
- `输出成果：` 描述交付物和验收标准。
