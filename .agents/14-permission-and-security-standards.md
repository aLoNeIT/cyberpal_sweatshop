# 权限与安全规范

本文档用于统一内部项目的登录态、接口权限、输入输出安全、文件安全和敏感信息保护要求。

## 1. 登录态与权限

- 新增接口默认必须纳入权限控制，除登录、验证码、健康检查、公开静态资源等明确公共接口外，不得绕过权限校验。
- 需要放行的公共接口必须加入明确白名单，并在代码注释或配置说明中写明放行原因。
- 业务接口应优先使用 `PermissionAnnotation` 声明权限码；未配置权限注解且未命中白名单的接口应默认拒绝访问。
- 权限码应按应用、模块、资源、操作组织，避免多个无关接口复用同一个权限码。
- 涉及跨主体、跨用户的数据访问时，必须同时校验登录用户的数据范围，不能只校验接口功能权限。

## 2. 输入过滤与参数校验

- 接收前端参数时必须先过滤、再校验、后使用；禁止直接把原始请求参数传入 `Model` 查询或写入数据库。
- 参数校验必须覆盖必填、类型、长度、枚举值、ID 是否为正整数、数组结构和文件上传约束。
- 查询条件必须通过白名单字段映射构造，禁止由前端直接传入数据库字段名、排序表达式或原始 SQL 片段。
- 批量操作必须限制单次数量，并校验每个 ID 或业务键是否属于当前用户可操作范围。
- 文件上传必须校验文件大小、扩展名、MIME 类型和业务场景；禁止仅依赖前端文件名判断类型。

## 3. 响应与脱敏

- 响应数据必须按接口场景最小化返回，禁止把模型原始数组无筛选地直接返回给前端。
- 手机号、身份证、银行卡、处方、支付、Token、密码、盐值、密钥、验证码等敏感字段不得明文返回。
- 脱敏逻辑必须使用统一脱敏函数或统一封装，避免各接口自行拼接造成规则不一致。
- 列表接口默认不返回大字段、审计字段、删除字段和内部实现字段；确需返回时必须在接口注释中说明用途。
- 下载、图片、文件流等二进制响应也必须先校验访问权限，再返回文件内容或地址。

## 4. 敏感操作

- 删除、禁用、审核、月结、锁定、退款、额度调整、密码重置、角色授权等敏感操作必须进行权限和数据范围双重校验。
- 敏感操作应记录操作人、操作时间、业务主键、操作前后关键状态和来源应用。
- 重要状态流转必须在 `Logic` 层校验前置状态，禁止只依赖前端按钮状态控制。
- 涉及外部系统、短信、支付、文件生成等副作用时，必须考虑失败后的补偿或重试策略。

## 5. 密钥与凭据

- 密码必须使用带盐或更高强度的哈希方式保存，禁止明文或可逆加密存储密码。
- Token、refresh token、验证码、临时授权码必须设置有效期，并避免在日志和响应中重复暴露。
- 第三方密钥、数据库密码、Redis 密码、短信密钥、证书私钥不得提交到仓库。
- 调试接口、测试账号、临时后门和绕过权限的代码不得进入正式分支；确需临时排查时必须在任务结束前移除。
